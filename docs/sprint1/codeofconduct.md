# Code of Conduct (Team Agreement)

## Communication
- Be respectful and professional in all messages and meetings.
- Use the team group chat as the main channel for updates and questions.
- If you are stuck or delayed, tell the team early (same day).
- Respond to messages within 24 hours.

## Work Process
- No one pushes directly to `main`. Use feature branches + pull requests.
- Branch naming: `feature/your-name-description` (e.g. `feature/john-price-model`).
- Keep commits small and descriptive (e.g. `Add Price model class`, not `update stuff`).
- Review at least one teammate's pull request before merging.

## Quality
- Keep code readable (clear names, comments where needed).
- Every file must have the correct package declaration and match the project structure.
- Do not break the build on purpose; test before pushing.

## Meetings & Decisions
- We hold two meetings per week on Wednesday and Thursday.
- Meetings are used to discuss progress, assign tasks and resolve blockers.
- All members are expected to attend; notify the team in advance if you cannot.
- Meeting notes (date, attendees, topics, decisions) must be saved in `docs/`.

## Sprint & Deadlines
- Each Sprint deadline must be treated as a hard deadline; no late submissions.
- Tasks for each Sprint are assigned at the start and tracked on the project board.
- If a task is at risk of being late, raise it in the next meeting immediately.

## Fair Contribution
- Tasks should be shared fairly and everyone should contribute commits.
- Each member must update the project board with their task progress.
- If conflicts happen, discuss calmly and agree on a solution as a team.
- If a member is repeatedly not contributing, the issue will be raised with the module leader.
